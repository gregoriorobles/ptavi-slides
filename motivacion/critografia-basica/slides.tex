\frametitle{Definiciones}

\begin{itemize}
\item Kryptósgráfo, o el arte de la ``escritura escondida''. Cómo esconder contenidos o la existencia de mensajes a un adversario

\item De manera informal, el cifrado muestra el contenido de un mensaje de manera ininteligible a los que no posean cierta información secreta.

\item Esteganografía, o ``escritura encubierta''. Técnica utilizada para ocultar la existencia de un mensaje -- ¡incluso uno a simple vista!

\end{itemize}

\begin{frame}
\frametitle{One-time pad}

\begin{itemize}
\item Se genera una clave aleatoria de tamaño tan grande como tu mensaje
y utilizas luego un OR exclusivo (XOR) entre la clave y el mensaje
\item Se trata de un método seguro e infalible (además, probado matemáticamente)
\item Pero...
 \begin{itemize}
  \item Para transmitir un secreto grande, necesitas transmitir primero un secreto grande
  \item La clave es tan grande como el mensaje
  \item Sólo se puede utilizar una vez
  \item Hay que tener una clave por mensaje y por receptor
 \end{itemize}
\end{itemize}

\end{frame}

\begin{frame}
\frametitle{Cifrado de clave pública (y II)}

Bonitas propiedades:
\begin{itemize}
  \item Una clave por persona
  \item Se puede comunicar con un desconocido sin acordar una clave.
\end{itemize}

Pero también tiene sus problemas (o retos):
\begin{itemize}
  \item ¿Esto es posible?
  \item ¿Cómo obtienes la llave pública de Alice?
  \item ¿Puedes fiarte del texto cifrado?
\end{itemize}

\end{frame}

\begin{itemize}
  \item Resumen de mensaje comprime la entrada en un string de longitud fija
  \item No necesita claves
  \item Es difícil encontrar la entrada a partir del hash
  \item Resistencia contra colisiones: es difícil encontrar dos entradas que generen el mismo hash.
